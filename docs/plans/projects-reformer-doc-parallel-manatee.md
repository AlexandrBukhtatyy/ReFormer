# Реструктуризация навигации reformer-doc: per-package сайдбары + шапка-переключатель пакетов

## Context

Сейчас у сайта один общий `docsSidebar` (Введение → Getting Started → Packages{все 6 пакетов вложенными категориями} → Паттерны → Примеры), а в шапке — `Docs | Packages ▾ | Examples | MCP | Core API | Playground`. Пользователь хочет:

1. **Боковая навигация — только по активному пункту шапки.** Кликнул «UI-Kit» → в сайдбаре только ui-kit; кликнул «Core» → только core. Никакого общего дерева со всеми пакетами.
2. **Шапка = список пакетов + Playground:** `Core | CDK | UI-Kit | Renderer | MCP | Playground` (+ локаль, GitHub справа). «Renderer» — один пункт, охватывающий **два** пакета (renderer-react и renderer-json).

Решения пользователя по сквозным разделам:
- **Базовые/сквозные** (Getting Started, Core Concepts, Validation, Behaviors, React, Паттерны, **Core API Reference**) → складываются в **сайдбар Core**. Отдельный пункт «Core API» из шапки убирается.
- **Витрина Examples** → доступна **ссылкой с главной**, отдельного пункта в шапке нет.

Механика Docusaurus: пункт навбара `type: 'docSidebar'` показывает сайдбар, к которому принадлежит текущий документ, и подсвечивается по активному сайдбару. Каждый документ может быть **ровно в одном** сайдбаре — поэтому разбивка = переназначение каждого дока в один из пяти сайдбаров.

## Целевая структура

### Навбар (`docusaurus.config.ts`, `themeConfig.navbar.items`)
Заменить текущие пункты (Docs / Packages-dropdown / Examples / MCP / Core API) на:
```ts
{ type: 'docSidebar', sidebarId: 'coreSidebar',     position: 'left', label: 'Core' },
{ type: 'docSidebar', sidebarId: 'cdkSidebar',      position: 'left', label: 'CDK' },
{ type: 'docSidebar', sidebarId: 'uiKitSidebar',    position: 'left', label: 'UI-Kit' },
{ type: 'docSidebar', sidebarId: 'rendererSidebar', position: 'left', label: 'Renderer' },
{ type: 'docSidebar', sidebarId: 'mcpSidebar',      position: 'left', label: 'MCP' },
{ href: '<stackblitz>', label: 'Playground', position: 'left' },
{ type: 'localeDropdown', position: 'right' },
{ href: '<github>', label: 'GitHub', position: 'right' },
```

### Сайдбары (`sidebars.ts`) — пять штук, каждый ведёт со своей landing-страницы пакета
- **`coreSidebar`**: `packages/core` (landing) → Getting Started (`getting-started/installation`, `quick-start`) → Core Concepts (`core-concepts/nodes`, `reactive-state`, категория «Схемы» = `core-concepts/schemas/*`) → Валидация (`validation/*`) → Behaviors (`behaviors/*`) → React (`react/hooks`, `react/custom-fields`) → Паттерны (`patterns/*`) → категория «Core API Reference» = содержимое бывшего `apiSidebar` (`api/index` + autogenerated `api/classes|enumerations|functions|interfaces|type-aliases`).
- **`cdkSidebar`**: `packages/cdk` → `cdk/form-array`, `cdk/form-wizard`, `cdk/form-field`.
- **`uiKitSidebar`**: `packages/ui-kit` → `ui-kit/overview` → категории «Текстовые поля» / «Выбор» / «Действия и раскладка» / «Составные» (как сейчас во вложенной ui-kit категории).
- **`rendererSidebar`** (два пакета): `packages/renderer-react` и `packages/renderer-json` двумя doc-пунктами (детальных под-страниц пока нет). Ведёт с `packages/renderer-react`.
- **`mcpSidebar`**: добавить `packages/mcp` (landing) первым → затем существующие `mcp/overview`, `quick-start`, `tools`, `prompts`, `examples`.

Удалить `docsSidebar` и `apiSidebar` (их содержимое перераспределено). Категории переиспользуют уже существующие вложенные структуры из текущего `docsSidebar` (просто вынести на верхний уровень нужного сайдбара).

### Сквозные страницы
- **`intro`** (`docs/intro.md`, slug `/docs`): включить первым пунктом `coreSidebar` («Введение»), чтобы не осиротела и осталась достижима.
- **`packages/index`** («Обзор пакетов»): больше не нужна (дропдаун Packages удалён, главная линкует прямо на landing-и). Убрать из навигации; проще всего — удалить файл `docs/packages/index.md` (и его RU-перевод, если появится) во избежание orphan-предупреждения.
- **`examples/index`**: убрать из сайдбара/шапки, оставить страницу. На **главной** ([src/pages/index.tsx](projects/reformer-doc/src/pages/index.tsx)) добавить видимую ссылку/карточку на `/docs/examples/` (сейчас Examples был только в навбаре). Кнопку «Get Started» в hero перенацелить на `/docs/packages/core` (т.к. пункта Docs больше нет).
