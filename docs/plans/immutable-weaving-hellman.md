# Закрепить snapshot form-schema.schema.json (вариант B)

## Context

Пользователь заметил: `projects/.../complex-multy-step-form-renderer-json/form-schema.schema.json`
«не расширяет схему из пакета renderer-json». Это **верно** — и by design, но с двумя
изъянами.

Как устроено сейчас:

- Каноническая мета-схема form-DSL живёт в пакете:
  [packages/reformer-renderer-json/src/schema/form-schema.schema.json](../../packages/reformer-renderer-json/src/schema/form-schema.schema.json)
  — `$id: https://reformer.dev/schemas/form-schema.schema.json`, общий
  `componentOp.pattern = ^\$component\(.+\)$`.
- `buildFormSchemaMetaSchema({ componentNames })`
  ([schema/index.ts:44](../../packages/reformer-renderer-json/src/schema/index.ts#L44))
  делает `JSON.parse(JSON.stringify(base))` и **inline-переписывает** один паттерн `componentOp`
  на enum имён компонентов. Это полная материализованная копия (snapshot), не `$ref`.
- Генератор [scripts/gen-form-json-schema.ts](../../projects/react-playground/scripts/gen-form-json-schema.ts)
  пишет результат в playground-файл; `json-schema.json` ссылается через
  `"$schema": "./form-schema.schema.json"`.

Сравнение построчно: playground-файл **идентичен** базовому 1:1, кроме `componentOp.pattern`.
Отсюда два изъяна:

1. **Конфликт `$id`** — у копии тот же `$id`, что у базы (два разных документа, один канонический id).
2. **Молчаливый дрейф** — если база изменится, snapshot протухнет; генератор не закреплён
   (нет npm-скрипта, нет CI-гейта, файлы untracked).

**Решение пользователя — вариант B:** оставить материализацию (она нужна ради IDE-подсветки
**имён** компонентов — `$component(Inputt)` краснеет прямо в редакторе; честный `$ref`-extends
этого не даёт, т.к. VSCode не поддерживает `$dynamicRef`), но устранить оба изъяна:
**уникальный `$id` + закреплённый пайплайн генерации от единого источника (база пакета)**.

«Расширение» фиксируется не через `$ref`, а через **провенанс**: схема всегда _порождается_
из базы функцией `buildFormSchemaMetaSchema`, а не пишется руками — и гарантируется, что она
не дрейфует.

> Радиус намеренно минимальный: пакет `@reformer/renderer-json` **не меняется**
> (генератор импортит его `dist`, любая правка API потребовала бы ребилда dist).

## Changes

### 1. Генератор: развести `$id` + пометить как generated

[projects/react-playground/scripts/gen-form-json-schema.ts](../../projects/react-playground/scripts/gen-form-json-schema.ts)

После `const concrete = buildFormSchemaMetaSchema({ componentNames })` (строка 35) —
постобработка перед записью:

```ts
const concrete = buildFormSchemaMetaSchema({ componentNames });

// Развести $id с базовой мета-схемой пакета (избежать duplicate-$id) + пометить как generated.
concrete.$id = 'https://reformer.dev/schemas/credit-application.schema.json';
concrete.title = 'Credit application form schema (M1, generated)';
concrete.description =
  'GENERATED by react-playground/scripts/gen-form-json-schema.ts from ' +
  'createCreditApplicationRegistry — DO NOT EDIT BY HAND. Materialized snapshot of the ' +
  'base form-DSL meta-schema (@reformer/renderer-json) with componentOp tightened to the ' +
  'registry enum. Re-run `npm run gen:form-schema` after changing the registry.';

writeFileSync(outPath, JSON.stringify(concrete, null, 2) + '\n', 'utf8');
```

Остальная логика (валидация `json-schema.json` через `validateFormSchema`, exit 1) — без изменений.

### 2. npm-скрипт + встроить в build

[projects/react-playground/package.json](../../projects/react-playground/package.json)
(симметрично существующему `generate:mocks`):

```jsonc
"scripts": {
  "generate:mocks": "npx tsx scripts/generate-mocks.ts",
  "gen:form-schema": "npx tsx scripts/gen-form-json-schema.ts",
  "build": "npm run generate:mocks && npm run gen:form-schema && tsc -b && vite build",
  ...
}
```

Так сборка playground всегда регенерит схему и падает (exit 1), если `json-schema.json` стал
невалиден против реестра.

### 3. CI-гейт «schema не протухла»

[.github/workflows/ci.yml](../../.github/workflows/ci.yml) — добавить шаг **после**
«Build @reformer/renderer-json» (строка 57; генератор импортит его `dist`, поэтому пакет должен
быть собран первым):

```yaml
- name: Check generated form-schema is up to date
  run: |
    npm run gen:form-schema -w react-playground
    git diff --exit-code -- 'projects/react-playground/src/pages/examples/complex-multy-step-form-renderer-json/form-schema.schema.json'
```

Если реестр изменили, а committed-снапшот не перегенерили → diff непустой → CI падает.
Дополнительно сам генератор валидирует `json-schema.json` (exit 1 при ошибке).

> **Риск-проверка (см. Verification):** генератор тянет `createCreditApplicationRegistry`,
> который импортит UI-компоненты (`.tsx`). Нужно подтвердить, что `npx tsx gen-form-json-schema.ts`
> отрабатывает в чистом node (CI), а не только под vite. Если реестр тянет browser-only код —
> шаг гейта придётся доработать (напр. экспортировать «чистый» список имён отдельно). Автор уже
> написал генератор в расчёте на tsx-запуск, так что вероятно работает — проверяем эмпирически.

### 4. Закоммитить артефакты (делает пользователь)

Сейчас untracked/uncommitted: `form-schema.schema.json`, `gen-form-json-schema.ts`,
изменённые `package.json`, `ci.yml`. CI-гейт работает только для committed-снапшота.
Коммит — **отдельным шагом по явной просьбе** (CLAUDE.md: git-коммиты только по запросу);
план их не коммитит, только готовит.
