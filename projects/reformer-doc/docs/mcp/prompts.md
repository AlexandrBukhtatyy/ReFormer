# Промпты (Prompts)

Промпты — это предустановленные шаблоны для типовых задач. AI использует их для выполнения сложных операций.

## create-form

Генерация формы по текстовому описанию.

**Аргументы:**
| Аргумент | Обязательный | Описание |
|----------|--------------|----------|
| `description` | Да | Описание формы на естественном языке |

**Пример запроса:**

```
Создай форму регистрации с email, паролем и подтверждением пароля
```

**Что генерируется:**
1. TypeScript интерфейс формы
2. Схема формы (FormSchema) с компонентами
3. Правила валидации (ValidationSchema)
4. React-компонент с хуками

**Рабочий процесс:**

AI последовательно использует инструменты:
1. `get_recommended_structure` — структура проекта
2. `generate_types` — TypeScript типы
3. `generate_schema` — схема формы
4. `generate_validation` — валидация
5. `generate_behavior` — поведения (если нужны)
6. `check_code` — проверка качества

## generate-step-form

Генерация многошаговой формы.

**Аргументы:**
| Аргумент | Обязательный | Описание |
|----------|--------------|----------|
| `description` | Да | Описание шагов формы |

**Пример запроса:**

```
Создай форму заявки на кредит с шагами:
1. Личные данные (ФИО, дата рождения)
2. Контакты (телефон, email)
3. Документы (паспорт, СНИЛС)
4. Подтверждение
```

**Особенности:**
- Разделение на шаги с отдельными схемами
- Навигация между шагами
- Валидация каждого шага отдельно

## manage-validation

Управление правилами валидации.

**Аргументы:**
| Аргумент | Обязательный | Описание |
|----------|--------------|----------|
| `task` | Да | Описание задачи |

**Примеры запросов:**

```
Добавь валидацию email для поля userEmail
```

```
Сделай поле phone обязательным только если выбран способ связи "телефон"
```

```
Добавь проверку что пароль и подтверждение совпадают
```

**Возможности:**
- Добавление встроенных валидаторов
- Создание кастомных валидаторов
- Асинхронная валидация
- Кросс-валидация между полями

---

## manage-behavior

Управление поведениями формы.

**Аргументы:**
| Аргумент | Обязательный | Описание |
|----------|--------------|----------|
| `task` | Да | Описание задачи |

**Примеры запросов:**

```
Добавь автоматический расчёт total = price * quantity
```

```
Скрой поле companyName если userType !== "company"
```

```
Копируй значение billingAddress в shippingAddress при включении чекбокса sameAddress
```

**Возможности:**
- `computeFrom` — вычисляемые поля
- `enableWhen` / `disableWhen` — условное включение
- `watchField` — реакция на изменения
- `copyFrom` — копирование значений
- `syncFields` — двусторонняя синхронизация
- `resetWhen` — сброс при условии

## debug-form

Отладка проблем с формой.

**Аргументы:**
| Аргумент | Обязательный | Описание |
|----------|--------------|----------|
| `code` | Да | Код формы с проблемой |

**Пример запроса:**

```
Почему моя форма не показывает ошибки валидации?

[код формы]
```

**Что проверяется:**
- Использование `useMemo` для `createForm`
- Правильность `useFormControl` и `useFormControlValue`
- Соответствие типов и схемы
- Импорты валидаторов
- Вызов `markAsTouched` при blur
- Вызов `validate()` перед отправкой
