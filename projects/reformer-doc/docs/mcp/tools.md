# Инструменты (Tools)

MCP-сервер предоставляет набор инструментов, которые AI вызывает автоматически при выполнении задач.

## Генерация кода

### generate_schema

Получение шаблона и правил для создания FormSchema.

**Параметры:**
| Параметр | Тип | Описание |
|----------|-----|----------|
| `formType` | string | Тип формы: "registration", "checkout", "settings" и др. |

**Что возвращает:**
- Маппинг типов на компоненты (string → Input, boolean → Checkbox, и т.д.)
- Маски для документов РФ (телефон, ИНН, СНИЛС, паспорт)
- Правила структуры схемы
- Примеры из реальных проектов

---

### generate_types

**Параметры:**
| Параметр | Тип | Описание |
|----------|-----|----------|
| `formType` | string | Тип формы для контекста |

**Что возвращает:**
- Правила именования интерфейсов
- Правила типизации (undefined вместо null, без index signatures)
- Примеры типов

## Проверка качества

### check_code

Анализ кода формы на ошибки и проблемы качества.

**Параметры:**
| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `code` | string | Да | TypeScript код для анализа |

**Что возвращает:**
- Оценка качества (0-100 баллов)
- Список ошибок с объяснениями
- Предупреждения
- Рекомендации по исправлению

**Проверяемые правила:**
- Правильность импортов валидаторов и поведений
- Наличие `value` в каждом поле
- Использование `useMemo` для `createForm`
- Вызов `markAsTouched` при blur
- Проверка типов перед отправкой

## Помощь в разработке

### explain_error

Объяснение ошибки ReFormer с примерами исправления.

**Параметры:**
| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `error` | string | Да | Текст ошибки или описание проблемы |

**Распознаваемые проблемы:**
- Поле не обновляется при вводе
- Валидация не срабатывает
- TypeScript ошибки типов
- Форма пересоздаётся при каждом рендере
- Cannot read property / undefined
- Array index out of bounds

---

### get_pattern

Получение примера паттерна использования.

**Параметры:**
| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `pattern` | string | Да | Название паттерна |

**Доступные паттерны:**
- `conditional-fields` — условное отображение полей
- `computed-fields` — вычисляемые поля
- `cross-field-validation` — валидация между полями
- `async-validation` — асинхронная валидация
- `array-fields` — работа с массивами
- `multi-step-form` — пошаговые формы
- `form-submission` — отправка формы

---

### get_recommended_structure

Рекомендуемая структура файлов проекта.

**Параметры:**
| Параметр | Тип | Описание |
|----------|-----|----------|
| `complexity` | string | Сложность: "simple", "medium", "complex" |

**Что возвращает:**
- Структура папок
- Разделение файлов (types, schema, validation, behavior)
- Соглашения именования
