# i18n / Локализация

`@reformer/renderer-json` **formatter-agnostic**: определяет только интерфейс `LocaleService` + фабрики-замыкания. ICU (`intl-messageformat`), i18next, markdown-рендерер подключает потребитель своей реализацией сервиса — у пакета нет i18n-зависимостей.

## Два пути (важно выбрать правильный)

Значение попадает в один из двух приёмников, и это определяет путь:

| Приёмник | Что можно | Путь |
|---|---|---|
| **Строковый проп** (`label`, `placeholder`, `title`, `aria-*`) | только `string`, **статично** | `$locale(key)` / `{ $locale, params }` |
| **Контент** (видимый текст, сообщения) | `string` **или** rich/JSX, **реактивно** | компонент `$component(I18n)` |

Причина: `componentProps` спредятся в компонент как есть — signal/ReactNode в `label` уронит его (`Objects are not valid as a React child`). Строковый путь резолвится один раз при конвертации (смена языка = пересборка дерева); реактивный путь читает сервис из React-контекста на этапе рендера (live-переключение).

## Сервис локализации

```ts
interface LocaleService {
  resolve(key: string, params?: Record<string, unknown>): string;    // строка (обязателен)
  render?(key: string, params?: Record<string, unknown>): ReactNode; // rich/markdown (опционален)
  keys?: readonly string[];                                          // validate-time проверка ключей
}
```

Фабрики (0-dependency):

```ts
import { createLocaleResolver, createLocaleService } from '@reformer/renderer-json';

// плоский каталог ключ→строка
createLocaleResolver({ 'fields.email.label': 'Email' });

// таблица ключ→(params)=>строка — параметры и склонение без ICU (стиль cdk createMessageResolver)
createLocaleService({
  'fields.min': (p) => `Минимум ${p?.count} символов`,
  'users.count': (p) => plural(Number(p?.count), ['пользователь', 'пользователя', 'пользователей']),
});
```

### ICU MessageFormat (подключает потребитель)

Стандарт для склонения по CLDR-правилам. Библиотека — `intl-messageformat`; сервис пишешь сам:

```ts
import { IntlMessageFormat } from 'intl-messageformat';

function createIcuLocaleService(catalog: Record<string, string>, locale: string): LocaleService {
  return {
    resolve: (key, params) =>
      key in catalog ? (new IntlMessageFormat(catalog[key], locale).format(params) as string) : key,
    keys: Object.keys(catalog),
  };
}
// catalog: { 'users.count': 'There {count, plural, one {is # user} other {are # users}}' }
```

### Markdown / rich (подключает потребитель через `render`)

```ts
import ReactMarkdown from 'react-markdown';

const service: LocaleService = {
  resolve: (k) => catalog[k] ?? k,
  render: (k, p) => <ReactMarkdown>{format(catalog[k] ?? k, p)}</ReactMarkdown>,
  keys: Object.keys(catalog),
};
```

## Путь 1 — строковые пропы (статично)

Без параметров — строковый оператор:

```jsonc
{ "value": "$model(email)", "component": "$component(Input)",
  "componentProps": { "label": "$locale(fields.email.label)" } }
```

С параметрами — **структурная форма** (объект, не оператор — без парсинга аргументов):

```jsonc
{ "value": "$model(password)", "component": "$component(Input)",
  "componentProps": { "label": { "$locale": "fields.min", "params": { "count": 8 } } } }
// → resolve("fields.min", { count: 8 }) → "Минимум 8 символов"
```

Параметры здесь — **литералы** (путь статичный). Реактивные model-параметры — через `I18n` (путь 2).

Регистрация сервиса — в реестре (для конвертера):

```ts
const registry = defineRegistry((reg) => {
  reg.locale(createLocaleService({ 'fields.min': (p) => `Минимум ${p?.count} символов` }));
});
```

## Путь 2 — реактивный компонент `I18n`

Для markdown/rich и/или реактивных параметров и live-переключения языка. Регистрируется как компонент, оборачивается `LocaleProvider`:

```tsx
import { I18n, LocaleProvider, createLocaleService, JsonFormRenderer } from '@reformer/renderer-json';

const services = { ru: createLocaleService(ruTable), en: createLocaleService(enTable) };

function App() {
  const [lang, setLang] = useState<'ru' | 'en'>('ru');
  const registry = useMemo(() => defineRegistry((reg) => {
    reg.component('I18n', I18n);
    // ...остальные компоненты
  }), []);
  return (
    <LocaleProvider service={services[lang]}>
      <JsonRendererProvider settings={{ registry, model }}>
        <JsonFormRenderer schema={schema} />
      </JsonRendererProvider>
    </LocaleProvider>
  );
}
```

```jsonc
// в схеме — контейнер-узел; values.count приходит сигналом и обновляется вживую
{ "component": "$component(I18n)",
  "componentProps": { "id": "users.count", "values": { "count": "$model(userCount)" } } }
```

`I18n` рендерит `service.render?.()` (markdown/JSX) либо `service.resolve()` (строка) — формат решает сервис. Перерисовывается при смене сигнала `values.count` и при смене `service` в `LocaleProvider` (переключение языка).

## Валидация

`validateFormSchema` проверяет ключи `$locale(...)` и структурной формы `{ $locale }` против `localeKeys` (из `service.keys` или явного списка) — опечатка ключа ловится до рендера. Нет каталога → проверка мягко пропускается (ключи свободные).

## Anti-patterns

- **markdown/JSX в строковый проп** — `label: "$locale(...)"` с сервисом, возвращающим ReactNode, уронит компонент. Rich-контент только через `$component(I18n)` (или свой `$component(RichText)`).
- **реактивный параметр в строковом `$locale`** — `{ $locale, params: { count: "$model(...)" } }` не обновится (путь статичный, значение заморожено при конвертации). Реактивные model-параметры → `I18n` с `values`.
- **`I18n` без `LocaleProvider`** — без провайдера сервис по умолчанию отдаёт сам ключ (fallback-to-key). Оберни поддерево в `LocaleProvider`.
- **ждать live-переключения от `$locale`** — строковый путь статичен; смена языка для label/placeholder требует пересборки (новый сервис в `reg.locale` + пере-render). Live только у `I18n`.
- **тащить ICU/markdown в схему** — форматтер живёт в реализации `LocaleService` у потребителя, не в JSON.
